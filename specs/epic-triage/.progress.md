## Learnings
- Schema file uses 2-space indentation consistently
- `epicState` added as sibling to `state` and `task` definitions
- `epicName` added as last property in `state.properties`
- Existing templates use `---` frontmatter with key-value pairs and `{{UPPER_CASE}}` for frontmatter, `{{description}}` for content placeholders
- Agent frontmatter follows pattern: name, description, model: inherit, color: <color>
- Existing agents use `<mandatory>` tags for enforced sections and consistent Communication Style block

- Reference files are included via `${CLAUDE_PLUGIN_ROOT}/references/` pattern
- Reference files style: `# Title` line 1, blank line, `> Used by: <command>.md` line 3, then `## Section` headings
- start.md uses numbered steps (Step 1, 2, 3, 4, 5) with Step 3.5 inserted for epic detection between Scan Existing Specs and Route to Action
